# mc-bedrock-update

Nightly cron script that checks for a new Minecraft Bedrock Dedicated Server release, downloads it if needed, then performs a safe update in-place.

Generated by Claude AI (Anthropic) based on human instructions. Released into the public domain — no rights reserved, free to use, modify, and distribute without restriction.

---

## Requirements

- bash
- wget
- unzip

---

## How It Works

- Runs as the `mc` user — will exit immediately if run as anyone else
- Probes the Mojang CDN directly to detect new versions by incrementing from the currently installed version
- If a newer version is found: backs up `worlds/`, `server.properties`, `allowlist.json`, and `permissions.json`, unzips the new server, then restores those files
- If the install fails, data is automatically restored from the backup
- Keeps the last 7 backups, deletes the downloaded zip immediately after install
- All data files (logs, backups, version tracking) live under `/home/mc`

## Setup

### 1. Place the script

Copy `update.sh` to `/home/mc/` and make it executable:

```bash
chmod +x /home/mc/update.sh
```

### 2. Seed the version file

Set the currently installed version so the script doesn't attempt a needless reinstall on first run:

```bash
echo "1.26.3.1" > /home/mc/.installed_version
```

Replace `1.26.3.1` with whatever version is actually running. To confirm:

```bash
journalctl -u bedrock.service | grep Version | tail -3
```

### 3. Add cron entries

Open root's crontab:

```bash
sudo crontab -e
```

Add the following — the server is stopped at 02:58, the update script runs at 03:00, and the server is restarted at 03:03, giving an 8-minute window:

```
58 2 * * * systemctl stop bedrock.service
0  3 * * * su -s /bin/bash mc -c '/home/mc/update.sh >> /home/mc/logs/mc-bedrock-update.log 2>&1'
3  3 * * * systemctl start bedrock.service
```

## Directory Structure

| Path | Purpose |
|---|---|
| `/home/mc/update.sh` | The script |
| `/home/mc/.installed_version` | Tracks the currently installed version |
| `/home/mc/.mc-update.lock` | Lock file — prevents concurrent runs |
| `/home/mc/logs/mc-bedrock-update.log` | Log output (written by cron redirect) |
| `/home/mc/backups/` | Timestamped world/config backups (last 7 kept) |
| `/home/mc/bedrock-server-latest.zip` | Temporary download path (deleted after install) |

## Manual Run

To trigger the script manually, run it as the `mc` user:

```bash
su - mc
/home/mc/update.sh
```

## Changing Game Mode

Stop the service, run the binary directly to get an interactive console, then use the `gamemode` command:

```bash
sudo systemctl stop bedrock.service
su - mc
/home/mc/bedrock_server
```

Wait for `Server started`, then type:

```
gamemode survival PlayerName
```

Or for all players at once:

```
gamemode survival @a
```

Then stop cleanly with:

```
stop
```

Restart the service (as a sudoer):

```bash
sudo systemctl start bedrock.service
```
